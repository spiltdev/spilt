## Core BPE (8 contracts)

The foundational layer: capacity-weighted streaming payment routing for AI agents via Superfluid GDA.

| Contract | Purpose |
|----------|---------|
| [`CapacityRegistry`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/CapacityRegistry.sol) | Task type + sink registry, commit-reveal capacity signals, EWMA smoothing |
| [`StakeManager`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/StakeManager.sol) | Stake/unstake/slash, concave capacity cap $\sqrt{\text{stake}/\text{unit}}$ |
| [`BackpressurePool`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/BackpressurePool.sol) | Superfluid GDA pool with capacity-weighted unit rebalancing |
| [`EscrowBuffer`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/EscrowBuffer.sol) | Overflow buffer when all sinks saturated, FIFO drain |
| [`Pipeline`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/Pipeline.sol) | Multi-stage pipeline composition with upstream congestion propagation |
| [`PricingCurve`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/PricingCurve.sol) | EIP-1559-style dynamic fees based on queue length |
| [`CompletionTracker`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/CompletionTracker.sol) | Statistical capacity verification with auto-slash |
| [`OffchainAggregator`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/OffchainAggregator.sol) | Batched EIP-712 attestation submission (83.5% gas savings) |

## Demurrage (2 contracts)

Time-decaying tokens that incentivize circulation over hoarding, layered with velocity tracking.

| Contract | Purpose |
|----------|---------|
| [`DemurrageToken`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/DemurrageToken.sol) | ERC-20 Super Token with configurable decay rate per epoch |
| [`VelocityMetrics`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/VelocityMetrics.sol) | Epoch-based monetary velocity and turnover rate tracking |

## Nostr relays (2 contracts)

NIP-compliant relay capacity signaling and economic sustainability for relay operators.

| Contract | Purpose |
|----------|---------|
| [`RelayCapacityRegistry`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/RelayCapacityRegistry.sol) | Relay registration with EIP-712 signed capacity attestations, EWMA smoothing |
| [`RelayPaymentPool`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/RelayPaymentPool.sol) | Anti-spam minimum + BPE-weighted streaming payment distribution |

See also: [NIP-XX specification](/docs/nip-xx) for the full Nostr relay economics protocol.

| Contract | Purpose |

| [`CrossProtocolRouter`](https://github.com/puraxyz/puraxyz/blob/main/contracts/src/CrossProtocolRouter.sol) | Unified routing across Superfluid, Lightning, and on-chain settlement |

## Deployment

Contracts are deployed on **Base Sepolia** (testnet). Deployment addresses are tracked in `contracts/deployments/base-sepolia.json`.

### Build and test

```bash
cd contracts
forge build
forge test
```

All 319 tests pass across 32 test suites. The test suite covers:

- Core BPE: capacity registration, EWMA, staking, escrow, pricing, completion tracking
- Demurrage: wrap/unwrap, decay mechanics, epoch advancement, velocity metrics
- Lightning: node registration, EIP-712 batch attestation, EWMA smoothing
- Platform: reputation scoring, cross-domain aggregation, stake discounts
- V2 Economy: factory deployment, nested pool hierarchy, quality scoring, velocity decay, urgency token TTL
- Fork tests against live Superfluid on Base Sepolia
